## The Basics of Infrastructure Automation

Key Terms:

- Imperative: Also known as procedural; an approach where the commands to produce a desired state are defined and executed in order
- Declarative: You define a desired state and the tool conforms the existing system to the model
- Artifacts: deployable elements coming out of your build process: compiled executables, OS packages, VM images, etc.

3 Areas of Infrastructure Testing:

- Unit: tests the smallest components possible
- Integration: tests a real running system
- Security: tests for both positive (system hardening) and negative (vulnerabilities) security attributes

Unit Testing

- Use formatters (syntax highlighters) to identify issues
- Linters enforce standards or rules

Integration and Security Testing

- A common tool for testing and auditing infrastructure is InSpec, which can also be used to assert CIS hardening standards.

From Artifacts to Systems

- Provisioning: the process of making a server ready for operation, including hardware, OS, system services, and network connectivity
- Deployment: the process of automatically deploying and upgrading applications on a server
- Orchestration: the act of performing coordinated operations across multiple systems
- Configuration Management (CM): an overarching term dealing with change control of system configuration after initial provisioning; often also applied to maintaining and upgrading application and application dependencies
- Convergent: CM automation that runs to converge systems on a desired state
- Idempotent: the ability to execute a CM procedure repeatedly and end up in the same state
- Immutable: deployments that are not intended to be changed after deployment; instead, the entire system is redeployed if needed
- Self Service: the ability for an end user to kick off one of these processes without having to go through other people
- Push: when a producer initiates an operation
- Pull: when a consumer initiates an operation

- CloudFormation for AWS
- Azure Resource Manager Templates for Azure
- Chef Provisioning for Chef
- HashiCorp Terraform
- Ubuntu Juju

Benefits of Vagrant

- Allows you to replicate a production environment locally
- API-driven VMs
- Identical provisioning
- Version control config
- Multi-machine friendly
- Automatic disk sharing and SSH setup

Vagrant requirements:

- [Virtualbox](https://www.virtualbox.org/wiki/Downloads)
- [Vagrant](https://www.vagrantup.com/downloads.html)

Convergence Provisioning

- Reaching the desired state over time
- Idempotent code can be run repeatedly without any unintended changes
- Convergent provisioning tools handle orchestration by tracking state in a master registry.
- Nodes report their system information to the registry.

Container Orchestration

- Kubernetes: an open-source platform for automating deployment, scaling, and operations of application containers across clusters of hosts, providing container-centric infrastructure.
- Pods are the smallest deployable units of computing that can be created and managed in Kubernetes.
- Deployments describe the desired state and the way to change the system.
