## Using system documentation

- ``--help`` is the fastest for command line options
- ``man <command>`` shows the section numbers of manuals
- ``man 1 printf`` shows section 1 for printf (shell commands)
- ``man 3 printf`` shows section 3 for printf (library calls)
- ``apropos`` searches within man pages (run ``sudo mandb`` first)
- ``apropos -s 1,8 <command>`` will filter to only show man pages 1 and 8, which are commands
- double tab after a command can help show a list of available commands

## Working with files and directories

- ``ls -al`` view all files in the current directory
- ``ls -al --full-time`` to view the detailed timestamp on a file
- Absolute paths always start with ``/``
- Relative paths start from the current directory
- Navigation shortcuts:
  - ``cd /`` # root directory
  - ``cd -`` # previous directory
  - ``cd ~`` # home directory
- Create files with ``touch <name>``
- Create directories with ``mkdir <name>``
- Copy files with ``cp <source> <destination>``

## Creating links

- Use ``stat`` to view information about a file
- From here you can view the inode number and the number of links
- All files start with 1 hard link
- Hard links must be on the same filesystem, and must be files (not folders)
- When all hard links are removed, data is deleted.
- Use ``ln <source> <name>`` to create a hard link in the current directory
- Use ``ln -s <source> <name>`` to create a soft link in the current directory
- Use ``readlink <filename>`` to view the path for a short link
- Changing the filename breaks a shortlink (shown in red)
- A relative path for the soft link is a workaround for this

## List, set, and change standard file permissions

- ``ls -l`` view owner and group on files in the current directory
- ``chgrp <group name> <filename>`` change the current group of a file
- ``chown <username> <filename>`` change the current owner of a file
- When using ``ls -l``, the first character shows the file type:
  - d directory
  - \- regular file
  - c character device
  - l link
  - s socket file
  - p pipe
  - b block device
- The next 9 characters are 3 groups of permissions for u (owner) g (group) o (others)
  - r read
  - w write
  - x execute
  - \- no permission
- Permission are evaluated from left to right, meaning it applies the first set of permissions you match
- ``chmod <permission> <file or directory>``
  - ``u+ g+ or o+`` then ``r w or x``
  - Example: ``u+rw`` adds read and write for the user permission
  - The same applies when using \- to remove, or \= to set exact
  - You can set multiple groups of permissions by separating them with a comma
  - Example: ``chmod u=rwx, g=r, o=r``
- Permissions can also be set with octal notation
  - Example: 0640 = -rw-r-----
  - In the example the first 0 is used for special permissions (suid 4, sgid 2, sticky bit 1)
  - suid: A file always executes as the user who owns the file, regardless of the user passing the command.
    - Capital S means suid is set with NO execute permission
    - Lowercase s means suid is set with execute permission
  - gid: can be set on a file or a directory;
    - if file then it is executed as the set group,
    - if directory then all files have their group ownership updated
  - sticky bit: restricts deletion to only the owner

## Search for files

- Find command syntax: ``find [/path/] [search parameters]``
- Find command examples:
  - ``find /usr/share/ -name '*.jpg'`` show all files in /usr/share/ that end in .jpg
  - ``find /lib64/ -size +10M`` show all files in /lib64 that are larger than 10 megabytes
  - ``find /dev/ -mmin -1`` show all files that have been modified in the last minute
  - ``find -mtime 2`` 24 hour periods
  - ``find -name "f*" -size 512k`` using the AND operator
  - ``find -name "f*" -o -size 512k`` using the OR operator
  - ``find -not -name "f*"`` using the NOT operator
  - ``find /var/log -perm -g=w ! -perm /o=rw`` example chaining permission requirements together
    - Looks for files in /var/log where the group has write perms, but other does not have read+write
  - When looking specifically for files use `-type f`
  - To specify a range of sizes use ``find -name "f*" -size 512k -size 1024k``
- Note: modified time = contents were changed, change time = metadata was changed

## Compare and Manipulate File Content

- Use `cat` to view small files and `tac` to view the lines reversed.
- Use `tail` to view the last 10 lines or use `-n <number>` to specify how many lines
- The opposite of tail is the `head` command.
- Transform text with `sed 's/old/new/g' <filename>`
  - `s` means search and replace
  - `g` means global
  - This command previews the changes, use `-i` to make the changes
  - `gi` to ignore case
  - `~` you can replace the forward slashes with tilde to deal with special characters
  - `sed -i '1,1000d' <filename>` delete the first 1000 lines from a file
- To extract parts of a file use `cut`
  - `-d` to specify delimiter
  - `-f 1` to get the first word on each line
- Use `uniq` to remove adjacent repeating lines
- Use `sort` to organize the file and make repeating values adjacent
- You can sort then pipe to uniq to return unique values.
- View differences between files use `diff <file1> <file2>`
  - Use `-c` to provide context of where the lines are different
  - Use `-y` or sdiff to compare files side by side
- `wc -l` to view the amount of lines in a file

## Pagers and Vi

- `less <filename>`
  - arrow keys to move up and down
  - `/<search term>` to search through the file
  - press `n` to find the next instance, shift + n to reverse
  - Use `-i` to ignore case
- `more` does the same thing with less features
- Vi
  - While vi is open, you can type a `number and G` to go to a specific line
  - `dd` to delete the line and add it clipboard
  - `P` to paste the line above the cursor

## Grep

- Search for a text string inside a file
- `-i` Ignore case
- `-r` Search all files in a directory
- `-v` Invert (look for the opposite)
- `-w` Match words
- `egrep` is grep that allows regex
- `egrep -c` provides a count

## Regex Symbols

- `^` matches the beginning of a line
- `$` matches the end of the line
- `.` match a single character
- `*` match 0 or more characters in front of the asterisk
- `+` match 1 or more characters in front of the plus
- `{}` with 1 number: match exactly the character in front, with 2 numbers: match min and max
- `[]` match any single character
- `()` groups regex
- `?` match 1 or 0 characters in front of the question mark
- `[^]` negated ranges or sets
- `|` match either or

## Archive

- Order: Archive > Compress > Backup
- Archive the files with `tar`
- `tar cf archive.tar file1` create an archive and add file1
- `tar rf archive.tar file2` append file2 to archive.tar
- `tar xf archive.tar -C /tmp/` extract an archive to /tmp/
- Make sure to use `sudo` to preserve ownership

## Compress

- Three default compression utilities are usually installed:
  - `gzip file1` or `gunzip file1.gz`
  - `bzip2 file1` or `bunzip file1.bz2`
  - `xz file1` or `unxz file1.xz`
- When extracting use `-C` to specify to location to put the extracted files
- Use `-k` to keep the input files
- Use `-l` to list contents
- `zip` does the same. Use `zip archive file1` or `unzip archive.zip`

## Backup

- `rsync` is the tool for remote synchronization
- Example `rsync -a Pictures/ nate@10.0.0.2:/home/nate/Pictures/`
- Running this command a second time will only update changes since the last command

## Disk Imaging

- Always unmount the disk before an image to prevent changes while the image is being made
- Example `sudo dd if=/dev/vda of=diskimage.raw bs=1M status=progress`
  - `if=` input file
  - `of=` output file
  - `bs=1M` block size
  - `status=progress` show the back progress
- When you want to restore, take the previous command and reverse `if` and `of`

## IO Redirection

- `>` redirect output
- `>>` append output
- `<` stdin
- `1>` stdout
- `2>` stderr
- You can chain pipes indefinitely
- For example pipe output to `sort`, then to `column` to organize text
- If you want to redirect both normal output and warnings, place `2>&1 at the end of the command`

## Working with SSL Certificates

- Openssl is the utility for creating certificates, specifically X.509 certificates.
- `openssl req` is the subcommand used to create a certificate signing request (CSR)
- Two helpful commands for creating requests:
  - `openssl req -newkey rsa:2048 -keyout key.pem -out req.pem`
  - `openssl req -x509 -newkey rsa:2048 -keyout key.pem -out req.pem`
- If you need a cert or CSR, use `req` options, if you already have those things and want to decode it, use `x509` options.
